- 2026-03-27 20:30: Fixed theme auto-change regression from f6eed4a9. Root cause: Ghostty has TWO levels of conditional state â€” app-level (`App.config_conditional_state.theme`) and per-surface (`Surface.config_conditional_state.theme`). The KVO refactor only called `ghostty_app_set_color_scheme` (app-level), but `App.updateConfig` applies each surface's OWN conditional state via `config.changeConditionalState(surface.config_conditional_state)`. Surfaces that miss `viewDidChangeEffectiveAppearance` (hidden tabs, detached bonsplit panels) kept stale per-surface state and resolved conditional themes with the wrong scheme. Fix: added `propagateColorSchemeToAllSurfaces()` on AppDelegate that iterates all terminal panels and calls `ghostty_surface_set_color_scheme` on each. Called from the KVO observer via `DispatchQueue.main.async` after `ghostty_app_set_color_scheme`. Zig-side dedup in `Surface.colorSchemeCallback` prevents redundant work for surfaces that also got `viewDidChangeEffectiveAppearance`. Verified by reading upstream Ghostty's equivalent: `BaseTerminalController.updateColorSchemeForSurfaceTree()` does the same per-surface sweep. Build verified.
